# Changes on 2026-02-23

This document records the full implementation of the pre-loop remediation work and the resulting loop integration.

## 1. Entrypoint and orchestration wiring

- `src/main.rs`
  - Added real CLI dispatch.
  - `run` now initializes `AnthropicProvider` and invokes `loop::run`.
  - Added explicit non-zero exits for provider/init and runtime failures.
- `src/loop.rs`
  - Implemented orchestrator with:
    - project context collection
    - planner invocation
    - per-step iterative editor/runner cycle
    - reviewer-based proceed/retry/abort decisions
    - `dry_run` execution path
    - enforcement of `max_iterations_per_step`
    - enforcement of `max_total_iterations`
  - Added `LoopReport` and detailed `LoopError` variants.

## 2. Path safety and sandbox hardening

- `src/schema.rs`
  - Kept lexical relative-path checks (empty, absolute, `..`).
  - Added symlink-aware guard using canonicalized existing ancestors to reject escape from project root.

## 3. Edit batch semantic validation

- `src/schema.rs`
  - Added validation errors and checks for:
    - duplicate `write_file` path in a batch
    - `write_file` and `replace_range` conflict on same path
    - overlapping `replace_range` line intervals on same path

## 4. Transactional edit application

- `src/runner.rs`
  - `apply_edits` now snapshots all touched files before mutation.
  - On any apply failure, all touched files are rolled back to original bytes.
  - Added rollback error reporting path.

## 5. ReplaceRange newline behavior

- `src/runner.rs`
  - Reworked range replacement handling to preserve:
    - CRLF vs LF newline style
    - original trailing newline behavior
  - Added non-UTF8 guard for `replace_range` targets.

## 6. Strict mode command behavior

- `src/runner.rs`
  - Strict fmt stage changed from mutating `cargo fmt --all` to non-mutating `cargo fmt --all --check`.

## 7. Loop limit enforcement

- `src/loop.rs`
  - Enforced both per-step and total iteration caps during runtime execution.

## 8. CLI argument validation

- `src/cli.rs`
  - Added parser validation for `--max-iters` to reject zero.
  - Converted `max_total_iterations` derivation to `saturating_mul(5)`.

## 9. Plan semantic validation

- `src/planner.rs`
  - Added `validate_plan` and new `PlanError::InvalidStep`.
  - Rejected invalid planner output at runtime when:
    - description is empty
    - files list is empty
    - file path is empty
    - file path is absolute
    - file path contains traversal (`..`)

## 10. LLM API error handling

- `src/llm.rs`
  - Disabled automatic HTTP status-as-error in request config.
  - Added explicit mapping of HTTP `>=400` into `LlmError::ApiError { status, body }`.
  - Kept transport failures mapped to `RequestFailed`.

## 11. Environment mutation race in tests

- `src/llm.rs`
  - Added test-only global mutex and RAII env guard around `ANTHROPIC_API_KEY` mutation.
  - Prevents parallel-test interference from shared process environment writes.
